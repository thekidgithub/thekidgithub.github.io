二 前端渲染部分 (非计网部分)
DOM树 -> CSS规则树 -> 渲染树 -> 布局（Layout/reflow）-> 绘制（paint） ->显示（display, GPU进程，合成图层）
回流/重排：尺寸、位置等变化，代价大于重绘，必然引起重绘
重绘：color visibility
性能优化策略：

  • transform 替代 定位 （减少回流）
  • visibility:hidden 替代 display:none(reflow -> repaint)
  • 减少查询节点的位置属性（会导致回流）、避免频繁操作DOM（使用fragment）
  • 动画(使用requestAnimationFrame、在absolute上使用动画)
  • 不使用table布局
  • script标签加上async or defer，或把script标签放底部(js进程和GUI进程互斥，阻塞DOM构建)(async解析完立即执行，defer解析完会在HTML解析完后执行)
  • 简化DOM和CSSOM树（压缩文件、减少选择器嵌套、在DOM树末端改变class）

补充：进程与线程
进程是 CPU 资源分配的最小单位，线程是 CPU 调度的最小单位，一个进程包含一个或多个线程
浏览器的多进程架构：主进程、第三方插件进程、GPU进程、渲染进程
渲染进程中：GUI渲染线程、JS引擎线程、事件触发线程、定时触发器线程、异步http请求线程

状态码

1xx: 中间状态，较少见
2xx: 成功处理（200 OK 204 No Content 206 Partial Content）
3xx: 重定向 (301 Moved Permanently 永久重定向 302 Found 临时重定向 304 Not Modified 缓存)
4xx: 客户端错误 (400 Bad Request 403 Forbidden 404 Not Found)
5xx: 服务器错误 （500 Internal Server Error 501 502 503）
