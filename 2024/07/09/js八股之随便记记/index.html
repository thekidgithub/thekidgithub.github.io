js八股之随便记记~

2024/7/9 八股 js

数据类型及判断

基本: number string boolean BigInt symbol null undefined
引用: Object(Object Array Function Math Date RegExp)

判断:

  • typeof(引用类型不具体，且误判null)
  • constructor(obj.constructor === Object) (不能判断null undefined 且constructor会被改写)
  • instanceof (查找原型链，见手写)
  • isArray isNan isFinite isPrototypeOf
  • Object.prototype.toString.call

0.1 + 0.2 ：二进制无限循环 -> 截断 -> 精度损失
栈内存和堆内存：引用类型地址位于栈内存，值位于堆内存

关于symbol: 具有唯一性，常用于对象的键中

symbol 允许我们创建对象的“隐藏”属性，代码的任何其他部分都不能意外访问或重写这些属性。

不会被枚举（for in,keys） 但是assign时会复制
全局symbol: Symbol.for()

闭包与内存

var let const 区别（块级作用域，变量提升，重复声明）
关于变量提升： var变量与函数声明（函数声明：function xx() 函数表达式: xx = function() {}）

console.log(a)
var a = 1;
var getNum = function () {
  a = 2;
}
function getNum() {
  a = 3
}
console.log(a)
getNum()
console.log(a)
//undefined 1 2
//2 是因为，a=3是函数声明，被提升到上面去了（wxg，踩过坑qwq）

闭包，各种输出题（结合setTimeout、使用var等等）
解决方案：var改let 应用IIFE（保存了全局作用域window和当前函数的作用域）等

内存泄漏（未被垃圾回收机制正确回收的未使用的对象）：

  • 闭包
function fn2(){
  let test = new Array(1000).fill('isboyjc')
  return function(){
    console.log(test)//引用了外部变量，故不会被回收
    return test
  }
}
let fn2Child = fn2()
fn2Child()
fn2Child = null//解决方法
  • 全局变量 （及时置空）
  • DOM引用(remove后置空)
  • 定时器，事件监听器，监听者模式
  • Map Set (使用Weak，强引用与弱引用)
  • console

垃圾回收（GC）(判断对象的可达性)
旧：引用计数（缺点：循环引用）
标记清除（缺点：内存碎片 解决：标记整理）
V8的优化：新老生代，并行回收，增量标记，懒性清理

对象、原型

拷贝（深，浅）(赋值：复制地址)
浅：Object.assign lodash:_.clone 展开（…） slice() concat()
深：lodash: _.deepClone 手写 JSON(不能处理函数和正则)

对象的原始值转换（toString, valueOf）：
字符串 toString -> valueOf 数值或其他： valueOf -> toString
(toString默认 [object Object], valueOf默认 对象本身0)

proto 与 prototype: 前者用于具体对象，后者为构造器属性
constructor: 在prototype上，若重写pro会丢失
Object.create(pro) 以pro为原型创建对象
（Object.create(null) 创建无原型的对象，用于“字典”）

异步 事件循环

浏览器事件循环
要点： js单线程，同步任务和异步任务，宏任务和微任务，任务队列
宏 -> 微 -> 渲染 -> 宏 。。。

Promise.resolve().then(()=>{
  console.log('Promise1')  
  setTimeout(()=>{
    console.log('setTimeout2')
  },0)
})
setTimeout(()=>{
  console.log('setTimeout1')
  Promise.resolve().then(()=>{
    console.log('Promise2')    
  })
},0)
// p1 s1 p2 s2

宏任务：sett, seti, setimmediate, script 微任务：promise.then process.nextTick(微任务优先级最高) queueMicrotask

Node事件循环

timer: sett,seti
check : setimmediate

console.log('start')
setTimeout(() => {
  console.log('timer1')
  Promise.resolve().then(function() {
    console.log('promise1')
  })
}, 0)
setTimeout(() => {
  console.log('timer2')
  Promise.resolve().then(function() {
    console.log('promise2')
  })
}, 0)
Promise.resolve().then(function() {
  console.log('promise3')
})
console.log('end')
//start=>end=>promise3=>timer1=>timer2=>promise1=>promise2

注意：浏览器和node区别： Node在所有timer执行完后才清空微队列，因此上面代码在浏览器执行，是 t1 -> p1 -> t2 -> p2
