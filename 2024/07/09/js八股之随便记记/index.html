js八股之随便记记~

2024/7/9 js 八股

数据类型及判断

基本: number string boolean BigInt symbol null undefined
引用: Object(Object Array Function Math Date RegExp)

判断:

  • typeof(引用类型不具体，且误判null)
  • constructor(obj.constructor === Object) (不能判断null undefined 且constructor会被改写)
  • instanceof (查找原型链，见手写)
  • isArray isNan isFinite isPrototypeOf
  • Object.prototype.toString.call

0.1 + 0.2 ：二进制无限循环 -> 截断 -> 精度损失
栈内存和堆内存：引用类型地址位于栈内存，值位于堆内存

闭包与内存

var let const 区别（块级作用域，变量提升，重复声明）
关于变量提升： var变量与函数声明（函数声明：function xx() 函数表达式: xx = function() {}）

console.log(a)
var a = 1;
var getNum = function () {
  a = 2;
}
function getNum() {
  a = 3
}
console.log(a)
getNum()
console.log(a)
//undefined 1 2
//2 是因为，a=3是函数声明，被提升到上面去了（wxg，踩过坑qwq）

闭包，各种输出题（结合setTimeout、使用var等等）
解决方案：var改let 应用IIFE（保存了全局作用域window和当前函数的作用域）等

内存泄漏（未被垃圾回收机制正确回收的未使用的对象）：

  • 闭包
function fn2(){
  let test = new Array(1000).fill('isboyjc')
  return function(){
    console.log(test)//引用了外部变量，故不会被回收
    return test
  }
}
let fn2Child = fn2()
fn2Child()
fn2Child = null//解决方法
  • 全局变量 （及时置空）
  • DOM引用(remove后置空)
  • 定时器，事件监听器，监听者模式
  • Map Set (使用Weak，强引用与弱引用)
  • console

垃圾回收（GC）
旧：引用计数（缺点：循环引用）
标记清除（缺点：内存碎片 解决：标记整理）
V8的优化：新老生代，并行回收，增量标记，懒性清理
