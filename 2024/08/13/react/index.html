生命周期

(这些生命周期方法是类组件的方法，在函数组件上没有，但是仍会经历这些生命周期)
创建时：
constructor(初始化state等) -> getDerivedStateFromProps(比较props 和 state来加一些限制条件，防止无用的state更新) -> render -> componentDidMount(已挂载，可以获取DOM)
更新时：
getDerivedStateFromProps -> shouldComponentUpdate(判断是否需要重新渲染组件) -> render -> getSnapshotBeforeUpdate(获取更新前的状态，如滚动位置等，更新后回复状态) -> componentDidUpdate
卸载:
componentWillUnmount

事件模型

react自身实现了一套事件机制，称为合成事件(一个合成事件可能由多个原生事件合成，比如blur , change , input , keydown , keyup等 , 合成为onChange)
所有的事件都应用事件委托，绑定在上层的react的root element上（而不是DOM自身，减少内存开销）（react16以前是绑定到document上）
真实的dom上的事件被单独处理,已经被react底层替换成空函数
执行顺序：先执行原生事件，然后冒泡到上层执行react事件
流程：事件合成（事件插件） -> 事件绑定 -> 事件触发

fiber架构

js线程和GUI渲染线程是互斥的，过去的渲染过程不可中断，造成性能问题
fiber: 一种react定义的数据结构，是fiber树结构的基本单位，是一个js对象，包含元素信息
特性：可以中断和恢复，分解渲染任务，增量渲染，如果有优先级更高的任务就先挂起
实现：链表，记录父子 nextSibling

JSX

jsx -> 通过babel转换为react.createElement
babel会判断组件的首字母，小写为原生标签，大写为组件
为什么不能返回多个根：组件最后会被转换为render函数，函数的返回值只能有一个

diff key

diff过程（子节点数组）
先建立索引表，然后遍历新节点
oldIndex：当前节点在老集合中的下标
maxIndex：在新集合访问过的节点中，其在老集合的最大下标（maxIndex = max(oldIndex)）
当oldIndex>maxIndex时，将oldIndex的值赋值给maxIndex
当oldIndex=maxIndex时，不操作
当oldIndex < maxIndex时，将当前节点移动到index的位置

key:diff算法用，确定唯一标识
（不能用index，可能造成错误复用，不能用random,每次渲染都会变）
