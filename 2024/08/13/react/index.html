生命周期

(这些生命周期方法是类组件的方法，在函数组件上没有，但是仍会经历这些生命周期)
创建时：
constructor(初始化state等) -> getDerivedStateFromProps(比较props 和 state来加一些限制条件，防止无用的state更新) -> render -> componentDidMount(已挂载，可以获取DOM)
更新时：
getDerivedStateFromProps -> shouldComponentUpdate(判断是否需要重新渲染组件) -> render -> getSnapshotBeforeUpdate(获取更新前的状态，如滚动位置等，更新后回复状态) -> componentDidUpdate
卸载:
componentWillUnmount

事件模型

react自身实现了一套事件机制，称为合成事件(一个合成事件可能由多个原生事件合成，比如blur , change , input , keydown , keyup等 , 合成为onChange)
所有的事件都应用事件委托，绑定在上层的react的root element上（而不是DOM自身，减少内存开销）（react16以前是绑定到document上）
真实的dom上的事件被单独处理,已经被react底层替换成空函数
执行顺序：先执行原生事件，然后冒泡到上层执行react事件
原因：

  • 跨浏览器/跨平台
  • 事件委托减少开销
  • fiber架构支持，事件优先级

fiber架构

js线程和GUI渲染线程是互斥的，过去的渲染过程不可中断，造成性能问题
fiber: 一种react定义的数据结构，是fiber树结构的基本单位，是一个js对象，包含元素信息
特性：增量渲染，时间切片，中断恢复，优先级队列，双缓存
fiber树通过虚拟DOM树构建

过程：

  1. render 阶段：从 vdom 转换成 fiber，并且对需要 dom 操作的节点打上 effectTag 的标记
  2. commit 阶段：对有 effectTag 标记的 fiber 节点进行 dom 操作，并执行所有的 effect 副作用函数。

const PrivateRoute = (props: IPrivateRouteProps) => {
    const { element } = props;
    const authToken = tokenStorage.getItem(); //判断token，有继续，没有跳转登录
    if (authToken) {
        return <>{element}</>;
    }
    return (
        <>
            <Navigate to="/login" />;
        </>
    );
};
// 高阶组件HOC，用于给需要登录才能访问的页面添加路由守卫
export const withPrivateRoute = (Component: React.ElementType) => {
    // eslint-disable-next-line @typescript-eslint/no-explicit-any
    const WrappedComponent = (props: any) => {
        return <PrivateRoute element={<Component {...props} />} />;
    };
    return WrappedComponent;
};

关于useState:
更新方式：直接赋值和回调函数
直接赋值不能重复操作，回调函数可以
附：定时器
错误（原因: useEffect闭包）

useEffect(() => {
  const timer = setInterval(() => {
    setCount(count + 1);
  }, 1000);
  return () => clearInterval(timer);
}, []);

正确

useEffect(() => {
  const timer = setTimeout(() => {
    setCount(count + 1);
  }, 1000);
  return () => clearInterval(timer);
}, [count]);//添加依赖项

useEffect(() => {
  const timer = setInterval(() => {
    setCount(count => count + 1);
  }, 1000);
  return () => clearInterval(timer);
}, []);//使用回调函数

useState具有批量更新机制（要点：更新队列，回调函数下一次渲染前state的状态是固定的 -> state闭包）

useContext：提供跨层传递的props，用于语言等全局设置
用法：外层套Context-Provider, 内层使用useContext

useEffect和useLayoutEffect: 都是初始化/更新后执行,layout是在绘制前执行,effect在绘制后
useLayoutEffect场景:在浏览器绘制前获取 DOM 元素的大小或位置，或者在浏览器绘制前修改 DOM

为什么hook不能在if内使用：
以state为例，state的内部实现大概如下：

let values = [];
let currentHook = 0;
function useState(initialState) {
  if (typeof values[currentHook] === 'undefined') values[currentHook] = initialState;
  let hookIndex = currentHook;
  function setState(nextValue) {
    values[hookIndex] = nextValue;
    ReactDOM.render(<App />, document.getElementById('root'))
  }
  return [values[currentHook++], setState];
}

usestate以数组的形式存放每一个state，重新渲染时需要把数组按顺序访问一遍，如果放在条件语句内，则可能出现顺序混乱

JSX

jsx -> 通过babel转换为react.createElement
babel会判断组件的首字母，小写为原生标签，大写为组件
为什么不能返回多个根：组件最后会被转换为render函数，函数的返回值只能有一个

diff key

diff过程（子节点数组）
vdom树通过reconcile（调和）转为fiber结构（链表）,老fiber和新vdom进行diff
两次遍历：
第一次顺序查找可直接复用的节点
第二次老节点建立map，进行可复用节点的移动，剩下的多删少增

key:diff算法用，确定唯一标识
（不能用index，可能造成错误复用，不能用random,每次渲染都会变）

性能优化

渲染时：shouldComponentUpdate、PureComponent、React.memo

useCallback和useMemo:
cb是缓存函数体的，函数只有在props更新才更新
用法：函数作为依赖项（useEffect）；函数作为React.memo的组件props
memo缓存函数执行的结果
（useMemo !== React.memo，后者用来包装一个组件，只有props更新才重新渲染）

懒加载：lazy函数 or Suspense组件
Suspense: fallback={} 显示后备组件
里面包真正渲染的组件
lazy： lazy(() => import(‘./MarkdownPreview.js’));

纯函数组件（无状态组件），相比有状态组件（class），内存占用小
